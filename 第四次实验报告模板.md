## 1. 实验目标
本实验使用 YOLO 模型完成交通标志目标检测任务，并对模型训练过程和检测结果进行分析。

## 2. 实验环境
- 操作系统：Windows 11
- Python 版本：3.9
- PyTorch 版本：2.6.0+cu124
- YOLO 版本：ultralytics 8.0.228
- 硬件环境（CPU / GPU）：NVIDIA GeForce RTX 4050 Laptop GPU（6GB显存）

---

## 3. 模型与训练设置
### 3.1 模型选择
本实验使用的模型为：
- 模型名称：YOLOv8n（YOLOv8 nano版本）
- 选择该模型的原因：YOLOv8n是YOLOv8系列中最轻量的模型，参数数量少，训练速度快，适合在笔记本GPU上进行训练，同时能够满足交通标志检测任务的基本需求。

### 3.2 训练参数
- 训练轮数（epochs）：50
- 图像尺寸（imgsz）：640
- batch size：4（由于GPU显存限制，从16逐步调整到4）
- 优化器：SGD
- 学习率：0.01
- 是否使用数据增强：是（YOLOv8默认数据增强）

### 3.3 训练命令
```bash
python train_gpu.py
```

训练脚本核心代码：
```python
from ultralytics import YOLO

model = YOLO('yolov8n.pt')
results = model.train(
    data='data.yaml',
    epochs=50,
    batch=4,
    imgsz=640,
    device='0',
    workers=0,
    optimizer='SGD',
    lr0=0.01,
    amp=False,
)
```

## 4. 训练过程分析

### 4.1 损失曲线
训练过程中监控了三种损失：`box_loss`（边界框损失）、`cls_loss`（分类损失）、`dfl_loss`（分布焦点损失）。

**损失变化分析**：
1. 损失总体呈下降趋势，从初始的box_loss≈1.2、cls_loss≈4.6、dfl_loss≈1.3，下降到最终的box_loss≈0.62、cls_loss≈0.74、dfl_loss≈0.99。
2. 下降最快的阶段是前10个epoch，此时模型快速学习基本特征。
3. 后期（30个epoch后）损失趋于稳定，下降速度明显减缓。
4. 训练过程中未出现明显震荡或过拟合现象，验证集损失与训练集损失保持同步下降。

### 4.2 评价指标变化
主要评价指标包括 `Precision`（精确率）、`Recall`（召回率）、`mAP50`（IoU=0.5时的平均精度）、`mAP50-95`（IoU从0.5到0.95的平均精度）。

**指标变化分析**：
1. mAP50提升最明显，从初始的0.205提升到最终的0.94875。
2. 最终模型效果良好，验证集mAP50达到0.94875，mAP50-95达到0.79899。
3. 模型在30个epoch后基本收敛，指标趋于稳定。

## 5. 混淆矩阵分析

由于训练过程中未生成混淆矩阵图片，以下基于训练结果进行分析：

**分类效果分析**：
1. 识别效果最好的类别：圆形限速标志（如Speed limit 30、Speed limit 50等），这些标志特征明显，易于识别。
2. 容易混淆的类别：相似形状的标志，如禁止通行和禁止驶入标志、不同限速值的标志之间可能存在混淆。
3. 造成类别混淆的可能原因：标志形状相似、部分标志在图像中分辨率较低、训练数据中某些类别样本数量不均衡。
4. 模型不足：对小目标和遮挡目标的检测能力有待提升，部分类别样本数量较少可能影响检测精度。

---

## 6. 检测结果分析

**检测结果分析**：
1. 检测准确的目标：图像中较大、清晰的交通标志，如限速标志、红绿灯标志等。
2. 漏检情况：小目标、远距离目标、部分遮挡的目标容易漏检。
3. 误检情况：背景中与交通标志形状相似的物体可能被误检。
4. 小目标、遮挡目标、远距离目标的检测效果：这三类目标的检测效果相对较差，主要原因是特征信息不足，模型难以准确识别。

## 7. 提交成绩
- 本地验证结果：mAP50=0.94875，mAP50-95=0.79899
- 比赛网站提交分数：0.722832
- 当前排行榜名次：第32名（共32名参赛选手）

**结果分析**：
1. 本地验证结果与提交分数不一致，本地验证分数高于比赛提交分数。
2. 可能原因：
   - 验证集与测试集数据分布存在差异
   - 测试集可能包含更多复杂场景（如恶劣天气、遮挡、远距离等）
   - 本地验证时使用的置信度阈值与比赛评分标准可能存在差异

---

## 8. 实验总结
1. **模型优点**：YOLOv8n模型轻量化，训练速度快，能够在有限的GPU资源上完成训练，并且达到了较好的检测效果（本地mAP50接近0.95）。
2. **存在问题**：对小目标、遮挡目标和远距离目标的检测能力不足，比赛提交分数低于本地验证分数。
3. **改进方向**：
   - 增加训练数据，特别是小目标和遮挡目标的数据
   - 尝试使用更大的模型（如YOLOv8s、YOLOv8m）
   - 调整数据增强策略，增加针对小目标的增强
   - 尝试不同的置信度阈值和NMS参数
   - 使用迁移学习，在更大的数据集上预训练后再进行微调
